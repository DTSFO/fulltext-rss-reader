# Security

This public repository contains example configuration only. Do not commit `.env` files, password hashes, session secrets, database credentials, personal feed lists, or exported appearance packages.

The application treats RSS URLs as untrusted input. Keep private-network blocking, redirect validation, response limits, and HTML sanitization enabled when adapting the project.

Report a vulnerability privately through the contact address on the maintainer's GitHub profile. Do not include live credentials or private feed content in a public issue.
